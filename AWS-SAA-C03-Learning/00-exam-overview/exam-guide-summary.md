# Exam Guide Summary

## Exam Code

`SAA-C03`

## Question Types

- Multiple choice
- Multiple response

## Scoring

- Scaled score: 100-1000
- Passing score: 720
- Some questions may be unscored and used by AWS for exam evaluation

## Official Domains

| Domain | Focus | Weight |
| --- | --- | ---: |
| Domain 1 | Design Secure Architectures | 30% |
| Domain 2 | Design Resilient Architectures | 26% |
| Domain 3 | Design High-Performing Architectures | 24% |
| Domain 4 | Design Cost-Optimized Architectures | 20% |

## What The Exam Really Tests

- Can you choose the right AWS service for a scenario?
- Can you design secure access to resources?
- Can you make systems highly available and fault tolerant?
- Can you improve performance without overengineering?
- Can you reduce cost without breaking requirements?
- Can you compare similar services under pressure?

## High-Value Exam Skills

- Read the scenario carefully.
- Identify hard requirements: RTO, RPO, latency, compliance, scale, cost.
- Eliminate services that do not meet the requirement.
- Prefer managed services when they reduce operations.
- Choose multi-AZ for high availability.
- Choose multi-Region only when the scenario requires regional resilience, global latency, or disaster recovery.
- Avoid expensive solutions when a cheaper managed service meets the same requirement.
